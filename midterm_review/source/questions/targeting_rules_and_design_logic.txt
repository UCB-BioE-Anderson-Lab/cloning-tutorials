slug: reverse-complement-basic
difficulty: medium
topic: Applying a sequence transformation to make hidden information visible
question: |
  What is the reverse complement of the DNA sequence:

  <pre>ATCGAGA</pre>
choices:
  A: <pre>3'-ATCGAGA-5'</pre>
  B: <pre>5'-ATCGAGA-3'</pre>
  C: <pre>5'-TAGCTCT-3'</pre>
  D: <pre>5'-TCTCGAT-3'</pre>
answer: D
explanation: |
  The reverse complement is found in two steps. First, take the complement of
  each base (A↔T, C↔G), so <pre>ATCGAGA</pre> becomes <pre>TAGCTCT</pre>.
  Then reverse that string so the opposite strand is written in the standard
  5'→3' orientation, giving <pre>TCTCGAT</pre>.

  Option D is the complement only (not reversed). Option A is the original
  sequence with flipped end labels, not the reverse complement. Option C is the
  unchanged input.

slug: protospacer-extraction-indexing
difficulty: medium
topic: Translating a biological positional rule into correct sequence indices
question: |
  A function searches a target DNA string for the Cas9 PAM and extracts
  the 20 bp protospacer. Given:

  <python>
  target = 'ATGCCCTAGATGCCTGGCTCAGAAACCTGCC'
  pam_index = target.find('GG', 21)   # returns 24
  </python>

  Which slice correctly extracts the protospacer?
choices:
  A: target[pam_index - 20 : pam_index]
  B: target[pam_index - 21 : pam_index - 1]
  C: target[pam_index - 20 : pam_index - 1]
  D: target[pam_index - 21 : pam_index]
answer: B
explanation: |
  For Cas9, the PAM is NGG and the N is not part of the protospacer. If
  <python>pam_index</python> points to the first G of GG, then the N is at
  <python>pam_index - 1</python>. The 20 bp protospacer ends immediately before
  that N.

  Therefore the protospacer spans indices <python>pam_index - 21</python>
  through <python>pam_index - 2</python> inclusive, which in Python slicing is
  <python>target[pam_index - 21 : pam_index - 1]</python>.

  Option B includes the PAM-adjacent N. Option C is only 19 bp. Option D is 21
  bp and also includes the N.

question: |
  A student's gRNA design function extracts the protospacer as:

  <python>
  protospacer = target[pam_index - 20 : pam_index]
  </python>

  instead of the correct <python>target[pam_index - 21 : pam_index - 1]</python>.
  The resulting guide RNA is synthesized and tested. What is the most likely
  outcome?
choices:
  A: The guide is shifted by one base toward the PAM and may overlap it, which often reduces or abolishes editing
  B: The guide works the same in practice because a one-base shift is usually tolerated during Cas9 targeting
  C: The code fails immediately with an IndexError because the slice end no longer matches the PAM boundary rule
  D: The designed guide is only 19 bases long, so synthesis fails before the targeting experiment is performed
answer: A
explanation: |
  The incorrect slice still returns 20 bases, but it includes the N of the NGG
  PAM and omits the first base of the true protospacer. That shifts the guide by
  one base toward the PAM.

  Cas9 targeting depends on the correct protospacer sequence adjacent to (not
  overlapping) the PAM, so this shift will often severely reduce or eliminate
  editing. Option C is wrong because the slice is syntactically valid. Option D
  is wrong because the slice length is still 20.

difficulty: medium
topic: Recognizing when a search must consider complementary representations of the same data
question: |
  A function searches for Cas9 PAM sites on a target gene:

  <python>
  def find_pam(target):
      return target.find('GG', 21)
  </python>

  It returns -1 for a known targetable gene. A colleague points out the
  error. What did the function fail to do?
choices:
  A: Search for CC on the forward strand so it can detect the complement of GG without computing strand reversal
  B: Use a regular expression search instead of .find() so degenerate PAM patterns can be matched automatically
  C: Start searching at index 0 rather than 21 so early PAMs are not missed in short coding sequences
  D: Search the reverse complement for GG so PAMs on the opposite strand are checked with the correct geometry
answer: D
explanation: |
  DNA is double-stranded, so a valid PAM can occur on either strand. If the
  forward strand search finds no valid GG in the required position range, a PAM
  may still exist on the opposite strand and will be found by searching the
  reverse complement.

  Option B is not equivalent, because simply searching for <python>'CC'</python>
  on the forward strand does not preserve the correct spatial relationship
  between PAM and protospacer. Option C does not fix the strand-coverage error.
  Option D changes the search window but not the missing-strand logic.

slug: no-valid-pam-error-handling
difficulty: medium
topic: Deciding what a function should do when a required precondition is not met
question: |
  A gRNA design function searches for a PAM in the target sequence but
  finds none. Which is the best way to handle this case?
choices:
  A: Return None without any message and rely on the caller to remember to check for missing designs later on
  B: Return a guide built from positions 0-19 so the pipeline continues even when the target lacks a valid PAM
  C: Print a warning and return an empty string so downstream steps can decide whether the result is usable
  D: Raise a ValueError with a clear message stating that no valid PAM was found in the provided target sequence

explanation: |
  If no PAM is present, the biological precondition for design is not met and
  the function cannot produce a valid guide. Raising a <python>ValueError</python>
  with a clear message makes the failure explicit and prevents downstream code
  from using an invalid design.

  Option B can be made to work in some coding styles, but silent return values
  are easy to misuse. Option C returns an invalid design. Option D mixes error
  reporting with ambiguous return values.

slug: strand-orientation-and-protospacer
difficulty: medium
topic: Reasoning about how strand orientation determines which 20 bp sequence is the protospacer
question: |
  A target gene's forward strand (5'→3') contains:

  <pre>
  ...ACGTTGACCGATCCTGAAGG TCA...
                       ^^^
                       NGG (PAM)
  </pre>

  The PAM on the forward strand is <pre>AGG</pre>. A colleague says the
  protospacer must always be taken from the reverse strand.

  Which statement is correct?
choices:
  A: The protospacer should always be taken from the reverse strand because Cas9 binds double-stranded DNA symmetrically
  B: Either strand can be used interchangeably because complementary strands encode the same targeting information
  C: The protospacer is the 20 bp immediately upstream of the PAM on the same strand where that PAM is identified
  D: The protospacer should be selected from whichever strand has higher GC content near the PAM for guide stability
answer: C
explanation: |
  For Cas9, the protospacer is defined relative to the PAM location: it is the
  20 bp sequence immediately upstream (5') of the NGG PAM on the same strand in
  which the PAM is identified. If the PAM is on the forward strand, the
  protospacer is read from the forward strand. If the PAM is on the reverse
  strand, the protospacer is read from the reverse strand.

  Option B is wrong because strand choice depends on PAM location. Option C is
  wrong because the two strands are complementary, not identical. Option D is
  unrelated to the targeting rule.
